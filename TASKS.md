### 📏 Легенда Story Points (SP)
* **1 SP**: Изи-бризи, написать класс, геттеры/сеттеры. (До 2 часов)
* **2 SP**: Простая логика, чтение файла, базовый метод. (Полдня)
* **3 SP**: Требует подумать, погуглить API, написать пару тестов. (День)
* **5 SP**: Сложная таска. Сеть, многопоточность, неочевидные баги ОС. (2-3 дня)
* **8 SP**: Эпик, надо разбивать.

## 📂 Epic 2: FileSync (Магическая папка)

| ID | Задача | Кому | SP | Описание (User Story & Tech) | Acceptance Criteria (DoD) |
|---|---|---|---|---|---|
| **SYNC-1** | FileWatcher | **Z** | 3 | **US:** Как юзер, я хочу, чтобы прога сама видела, что я закинул файл в папку.<br>**Tech:** Юзать `java.nio.file.WatchService`. Слушать `ENTRY_CREATE` и `ENTRY_MODIFY`. | - Игнорит временные файлы (типа `~$word.doc` или `.tmp`).<br>- Дергает коллбэк (интерфейс) при появлении файла. |
| **SYNC-2** | Хеширование (SHA-256) | **Z** | 2 | **US:** Как система, я хочу знать уникальный отпечаток файла для дедупликации.<br>**Tech:** `MessageDigest.getInstance("SHA-256")`. Читать файл батчами (буфером), чтобы не выжрать RAM на больших файлах. | - Метод `String getHash(Path file)` работает и для 1 КБ, и для 2 ГБ.<br>- Написан Unit-тест. |
| **SYNC-3** | Протокол TCP | **A** | 2 | **US:** Как разработчики, нам нужен стандарт передачи файла.<br>**Tech:** Придумать структуру. Например: Сначала летит 4 байта длины JSON хедера. Потом сам JSON (имя файла, размер, хеш). Потом сырые байты файла. | - Описана дока/record-модели протокола. |
| **SYNC-4** | TCP Сервер | **A** | 5 | **US:** Как нода, я хочу принимать файлы от бро.<br>**Tech:** `ServerSocket` -> `accept()` -> спавним тред. Читаем хедер, если хеш совпадает с тем, что уже есть (скачали ранее) — дропаем соединение. Иначе пишем файл кусками в папку. | - Файл сохраняется.<br>- Избегаем OutOfMemory (пишем чанками). |
| **SYNC-5** | TCP Клиент | **A** | 5 | **US:** Как нода, я хочу отправлять файл всем, кто онлайн.<br>**Tech:** При срабатывании `FileWatcher` берем список IP из `PeerRegistry` и через `ExecutorService` (тредпул) параллельно шлем файл каждому. | - Файл успешно улетает 2+ пирам параллельно. |
| **SYNC-6** | Защита от эха (Дедупликация)| **A** | 3 | **US:** Как система, я не хочу отправлять файл обратно тому, от кого его только что получил (луп).<br>**Tech:** Кэш хешей недавно принятых файлов. Если `WatchService` видит новый файл, сначала проверяем, не мы ли его только что скачали. | - Бесконечный цикл пересылки одного файла не возникает. |

> **👨‍🏫 Менторская заметка:** На SYNC-1 Z может столкнуться с тем, что `WatchService` срабатывает дважды на один файл (сначала пустой, потом наполненный виндой). Посоветуй ему юзать задержку (Debounce) или проверять `Files.isReadable()` и блокировку файла.
